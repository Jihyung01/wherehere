# 프로필 피드·로그인 유도·개인정보 수정 변경사항

## 1. 로그인 유도 (시작 화면)

- **비로그인** 사용자가 앱에 들어오면 **로그인 유도 화면**이 먼저 표시됩니다.
- "로그인하고 나만의 추천을 받아보세요" 문구 + **로그인/회원가입** 버튼(주요 CTA).
- **데모로 체험하기** 버튼으로 데모 모드 진입 가능(같은 탭 세션 동안 유지).

## 2. 개인정보 탭 (설정 > 개인정보)

- **프로필 URL·닉네임 수정 시 튕김 현상 수정**: 개인정보 펼침 영역에 `onClick` 전파 차단(`stopPropagation`)을 적용해, 입력란 클릭/포커스 시 섹션이 접히지 않도록 했습니다.
- 표시 이름·프로필 이미지 URL은 **onBlur 시** `POST /api/v1/social/profile`로 저장됩니다.

## 3. 프로필 페이지 (인스타 스타일)

- **프로필 헤더**: 아바타, 닉네임(displayName), 아이디(친구 코드), "설정에서 프로필 수정" 링크.
- **기존**: 레벨/XP, 연속일·방문·퀘스트, 뱃지, PersonalityProfile.
- **내 피드**: 작성한 로컬 게시글을 **3열 그리드**로 표시(인스타 프로필 피드 스타일).
- 그리드 셀 클릭 → **게시글 상세 모달**: 본문, 이미지, 장소, **댓글 목록** + **댓글 입력·등록**.

## 4. API

- **GET /api/v1/local/posts?scope=user&author_id={userId}**  
  해당 사용자가 작성한 게시글만 조회(프로필 피드용).
- 기존 **GET /api/v1/local/posts/{post_id}/comments**, **POST /api/v1/local/posts/{post_id}/comments** 그대로 사용.

## 5. SQL

- **추가로 실행할 SQL 없음.**  
  기존 `local_posts`(author_id 컬럼), `local_comments` 테이블만 사용합니다.  
  이미 적용된 마이그레이션으로 충분합니다.
