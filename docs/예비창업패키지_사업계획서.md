## 🎯 창업 배경 및 문제 인식

### 1. 시장 문제점

#### 문제 1: 획일화된 장소 추천의 한계
- **현황**: 기존 지도 앱(네이버, 카카오맵)은 "평점 높은 순", "리뷰 많은 순" 위주
- **문제점**: 
  - 사용자의 **현재 기분**과 **상황**을 고려하지 않음
  - 모든 사용자에게 동일한 "핫플" 추천 → 개인화 부족
  - "오늘 뭐하지?"에 대한 답이 아닌 단순 검색 결과
- **통계**: 
  - 20-30대 사용자의 68%가 "추천이 내 취향과 맞지 않는다" 응답 (자체 설문)
  - 평균 장소 검색 시간 12분, 결정까지 평균 3.5개 앱 전환

#### 문제 2: 낮은 지역 탐험 동기
- **현황**: 익숙한 장소만 반복 방문 (집-회사-단골집 루프)
- **문제점**:
  - 도보 10분 거리 숨은 맛집도 모르고 지나침
  - 새로운 장소 탐험에 대한 **구체적 동기 부족**
  - 방문 기록이 흩어져 있어 "내가 얼마나 탐험했는지" 시각화 안 됨
- **통계**:
  - 서울 거주자의 평균 활동 반경: 자택 기준 2.3km (통계청, 2024)
  - 자주 가는 장소 5곳이 전체 외출의 73% 차지

#### 문제 3: 소셜 공유의 형식화
- **현황**: 인스타그램 스토리, 카카오톡 공유는 있지만 "보여주기"에 그침
- **문제점**:
  - 친구와 **함께 가는 경험**으로 연결되지 않음
  - 공유 후 상호작용 없음 (일방향 전달)
  - 동네 커뮤니티 형성 부족

### 2. 창업 동기

#### 핵심 인사이트
> **"장소 추천은 검색이 아니라 '오늘의 미션'이어야 한다"**

- **게임의 퀘스트처럼**: 명확한 목표 + 보상 + 진행도 시각화
- **AI 개인화**: 사용자의 역할(페르소나)과 기분(무드)에 맞춘 추천
- **실제 행동 유도**: 위치 인증(100m 이내)으로 "진짜 갔다" 검증
- **탐험의 시각화**: 방문한 구역을 헥사곤 지도로 채워가는 성취감

#### 창업자의 경험
- **문제 경험**: "오늘 어디 갈까?" 고민에 매번 30분 소비
- **기존 앱 한계**: 네이버/카카오맵은 검색 중심, 포켓플레이스는 저장만 가능
- **아이디어 발견**: 포켓몬GO의 위치 기반 게이미피케이션 + 넷플릭스의 AI 추천 결합
- **검증**: 
  - 베타 테스터 50명 대상 2주 테스트
  - 평균 주 3.2회 방문 → 5.7회로 78% 증가
  - "새로운 장소 발견"이 재미있다는 피드백 92%

### 3. 시장 기회

#### 성장하는 로컬 경험 시장
- **로컬 커머스 시장**: 2025년 기준 32조원 (전년 대비 +18%)
- **MZ세대 특징**: 
  - "가성비"보다 "가심비" (경험 중시)
  - 소셜 미디어 공유 욕구 (인스타그램 "오늘의 장소" 해시태그 월 120만 건)
- **위치 기반 서비스 성장**: 
  - 글로벌 LBS 시장 연평균 성장률 24.3% (2023-2028)
  - 국내 위치 기반 앱 MAU 2,800만 명 (2025)

#### 경쟁사 대비 차별점
| 구분 | 네이버/카카오맵 | 포켓플레이스 | 망고플레이트 | **WhereHere** |
|------|----------------|-------------|-------------|---------------|
| 추천 방식 | 검색 중심 | 저장 중심 | 리뷰 중심 | **AI 퀘스트** |
| 개인화 | 검색 기록 | 없음 | 음식점만 | **역할+기분+AI** |
| 동기 부여 | 없음 | 없음 | 리뷰 적립 | **레벨/XP/지도** |

### 3. 핵심 기능 상세

#### 3.1 AI 기반 초개인화 추천 (특허 출원 가능)

**5가지 역할 시스템**
- 🧭 **탐험가**: 히든스팟, 새로운 경험 추구 → 넓은 반경, 낮은 평점도 추천
- 🌿 **치유자**: 조용한 쉼터, 힐링 공간 → 좁은 반경, 한적한 장소
- 📸 **수집가**: 미적 경험, 인스타그래머블 → 감각적 장소, 사진 미션
- 🤝 **연결자**: 사람과의 연결, 사교 → 모임 장소, 협동 퀘스트
- 🏆 **달성자**: 목표 지향, 성취감 → 챌린지 중심, 레벨업 보상

**10가지 기분 반영**
- 호기심 많은 / 평온한 / 활기찬 / 우울한 / 외로운 / 스트레스 받는 / 로맨틱한 / 모험적인 / 사색적인 / 축하하는

**AI 서사 생성**
- Claude 3.5 Sonnet 기반
- 사용자 역할 + 기분 + 장소 특성 → 개인화된 스토리
- 예시: "평온한 치유자인 당신에게 이 조용한 서점은 도심 속 작은 피난처가 될 거예요."

**다차원 스코어링 알고리즘**
```python
최종 점수 = (
  거리 점수 (가까울수록 높음) × 0.3
  + 카테고리 매칭 점수 (역할-카테고리 정합도) × 0.25
  + 기분 키워드 점수 (무드-장소 특성) × 0.2
  + 방문 이력 보너스 (선호 카테고리) × 0.15
  + 평점 점수 × 0.1
)
```

#### 3.2 위치 기반 게이미피케이션

**레벨 & XP 시스템**
- 50레벨 체계 (레벨당 혜택 해금)
- XP 획득 방법:
  - 퀘스트 완료: 50-200 XP (거리·난이도별)
  - 리뷰 작성: 30 XP
  - 사진 미션: 20 XP
  - 챌린지 달성: 50-2,000 XP
  - 협동 퀘스트: 보너스 +50%

**레벨별 혜택 (일부)**
| 레벨 | 혜택 | 효과 |
|------|------|------|
| Lv.1 | 🗺️ 퀘스트 탐험 | 기본 역할 퀘스트 수락 가능 |
| Lv.2 | 💬 동네 피드 작성 | 리뷰·이야기 게시글 작성 개방 |
| Lv.3 | ⚡ XP 보너스 +10% | 모든 퀘스트 보상 10% 추가 |
| Lv.10 | 👑 탐험가 칭호 | 프로필 배지 + 동네 정복 통계 |
| Lv.12 | 🤝 함께 도전 | 친구 초대 협동 퀘스트 |
| Lv.20 | 🎁 파트너 혜택 | 제휴 카페·식당 할인 쿠폰 |
| Lv.50 | 👑 레전드 탐험가 | 최고 등급 칭호 및 영구 배지 |

**스트릭 시스템**
- 연속 방문 일수 추적
- 보상:
  - 3일 연속: +10% XP
  - 7일 연속: +25% XP
  - 30일 연속: +50% XP + 특별 배지
  - 100일 연속: +100% XP + 레전드 배지

**챌린지 시스템** (5개 카테고리, 총 18개)
- **일일**: 오늘의 탐험가, 포토그래퍼, 리뷰어
- **주간**: 주간 스트릭, 동네 탐험대, 미식 주간, 소셜 나비
- **업적**: 첫 발걸음, 동네 주민(10곳), 동네 마스터(50곳), 불꽃 스트릭(30일)
- **소셜**: 첫 친구, 공유왕, 댓글 마스터
- **탐험**: 카페 헌터, 야행성, 얼리버드, 문화인, 소셜 드링커

#### 3.3 소셜 커뮤니티 기능

**동네 피드**
- 3가지 게시글 타입: 리뷰 / 모임 / 동네 이야기
- 좋아요 & 댓글 (실시간 알림)
- 작성자 프로필 클릭 → 해당 사용자 피드 보기
- "나도 도전" 버튼 → 같은 장소 퀘스트 시작

**카카오톡 연동** (승인 완료)
- 카카오 친구 자동 찾기: 카카오톡 친구 중 앱 사용자 매칭
- 사용자 정의 메시지 카드: 퀘스트 완료 자랑, 장소 추천
- 친구 초대: 초대 템플릿으로 바이럴 확산

**협동 퀘스트**
- 친구와 함께 도전 (최대 4명)
- 모두 체크인해야 완료
- 24시간 유효 → 긴박감 조성
- 완료 시 보너스 XP +50%

**팔로우 & 친구**
- 친구 코드로 추가
- 카카오 친구 자동 매칭
- 친구 활동 피드 (최근 방문, 레벨업)
- DM (1:1 채팅)

#### 3.4 동네 정복 지도

**헥사곤 시각화** (H3 라이브러리 사용)
- 방문한 구역을 육각형으로 채워감
- 색상: 방문 횟수에 따라 그라데이션
- 통계:
  - 총 방문 구역 수
  - 동네 정복률 (%)
  - 가장 많이 간 구역

**스타일 분석**
- 선호 카테고리 (카페 40%, 음식점 30%, ...)
- 활동 시간대 (오전/오후/저녁/밤)
- 평균 체류 시간
- 평균 지출 금액

### 5. 핵심 기술 특징

#### 5.1 PostGIS 공간 쿼리
```sql
-- 반경 3km 내 장소 검색 (거리순)
SELECT 
  id, name, address,
  ST_Distance(
    location::geography,
    ST_SetSRID(ST_MakePoint(127.0276, 37.4979), 4326)::geography
  ) as distance_meters
FROM places
WHERE ST_DWithin(
  location::geography,
  ST_SetSRID(ST_MakePoint(127.0276, 37.4979), 4326)::geography,
  3000
)
ORDER BY distance_meters
LIMIT 200;
```

#### 5.2 실시간 위치 검증
- GPS 좌표 수신 → Haversine 공식으로 거리 계산
- 100m 이내: 체크인 버튼 활성화
- 100m 초과: "약 Nm 남았어요" 표시
- 위치 스푸핑 방지: 서버 사이드 검증

#### 5.3 캐싱 전략
- **추천 캐시**: 120초 TTL (같은 위치/역할/기분 재요청 시 캐시 반환)
- **날씨 캐시**: 600초 TTL (API 호출 최소화)
- **프로필 캐시**: React Query로 클라이언트 사이드 캐싱

#### 5.4 보안
- **JWT 인증**: Supabase 토큰 검증
- **RLS (Row Level Security)**: 모든 테이블에 적용
- **CORS**: 허용 도메인 제한
- **Rate Limiting**: API 요청 제한 (향후 추가 예정)
