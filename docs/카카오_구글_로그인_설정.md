# 카카오 · 구글 로그인 설정 가이드

## 1. 프론트엔드 (이미 구현됨)

- **로그인 페이지**: `/login` → 이메일 비밀번호 + **카카오로 계속하기** / **Google로 계속하기** 버튼
- **OAuth 콜백**: `/auth/callback` → Supabase가 로그인 후 리다이렉트하는 URL. 코드를 세션으로 바꾼 뒤 `/onboarding`(미완료 시) 또는 `/`(완료 시)로 보냄

필요한 건 **Supabase 대시보드에서 provider 설정**만 하면 됩니다.

---

## 2. Google 로그인 (Supabase 기본 지원)

### 2.1 Google Cloud Console

1. [Google Cloud Console](https://console.cloud.google.com/) → 프로젝트 선택 또는 생성
2. **API 및 서비스** → **사용자 인증 정보** → **사용자 인증 정보 만들기** → **OAuth 2.0 클라이언트 ID**
3. 애플리케이션 유형: **웹 애플리케이션**
4. **승인된 리디렉션 URI**에 아래 추가:
   - `https://<프로젝트참조ID>.supabase.co/auth/v1/callback`
   - Supabase 대시보드 → **Authentication** → **URL Configuration**에서 정확한 URL 확인
5. **클라이언트 ID**와 **클라이언트 보안 비밀** 복사

### 2.2 Supabase

1. **Authentication** → **Providers** → **Google** 켜기
2. **Client ID**, **Client Secret** 붙여넣기 → Save

이후 로그인 화면에서 "Google로 계속하기" 클릭 시 Google 로그인 → 콜백 → 세션 생성됩니다.

---

## 3. 카카오 로그인 (Supabase 사용 시)

Supabase는 **Kakao를 기본 Provider로 지원**합니다.

### 3.1 Kakao Developers

1. [Kakao Developers](https://developers.kakao.com/) → 내 애플리케이션
2. 앱 선택 → **앱 키**에서 **REST API 키** 확인
3. **카카오 로그인** → **활성화** ON
4. **Redirect URI** 등록:
   - `https://<프로젝트참조ID>.supabase.co/auth/v1/callback`
   - 로컬: `http://127.0.0.1:54321/auth/v1/callback` (Supabase 로컬 사용 시)
5. **동의 항목**: 닉네임, 프로필 사진 등 필요한 항목 설정
6. **코드** 생성 시 **Client Secret** 있으면 복사 (선택)

### 3.2 Supabase

1. **Authentication** → **Providers** → **Kakao** 켜기
2. **Kakao Client ID**: Kakao 앱의 **REST API 키** (앱 키에 있는 숫자)
3. **Kakao Client Secret**: Kakao 로그인 설정에서 발급한 Client Secret (있으면 입력)
4. Save

---

## 4. Redirect URL (필수)

Supabase **Authentication** → **URL Configuration**:

- **Site URL**: 서비스 메인 주소 (예: `https://wherehere-seven.vercel.app`)
- **Redirect URLs**에 다음 추가:
  - `https://wherehere-seven.vercel.app/auth/callback`
  - `http://localhost:3000/auth/callback` (로컬 테스트)

카카오/구글의 Redirect URI는 반드시 Supabase 콜백 URL(`.../auth/v1/callback`)로 두고, 그 다음 Supabase가 우리 앱의 `/auth/callback`으로 넘깁니다.

---

## 5. 환경 변수 (프론트)

이미 사용 중이면 변경 없음:

- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`

---

## 6. 동작 확인

1. `/login` 접속
2. **카카오로 계속하기** 또는 **Google로 계속하기** 클릭
3. 각 제공자 로그인 완료 후 `https://.../auth/callback?code=...` 로 돌아와야 함
4. 이후 `/onboarding` 또는 `/` 로 이동하면 성공

오류가 나면 브라우저 콘솔과 Supabase **Authentication** → **Logs**에서 확인하세요.
