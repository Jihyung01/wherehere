# 카카오톡 친구 목록 / 메시지 API — 설정 및 심사용 캡처 가이드

카카오 개발자 심사에서 **친구 목록·메시지** 사용 시 아래 테스트 이력이 필요합니다.

1. **카카오 로그인**
2. **'카카오톡 서비스 내 친구목록' 동의항목 동의 받기**
3. **친구 목록 가져오기**
4. **친구에게 메시지 전송**

---

## 1. 카카오 개발자 콘솔 설정

1. [Kakao Developers](https://developers.kakao.com) → 내 애플리케이션
2. 해당 앱 선택 → **앱 키**에서 REST API 키 확인
3. **제품 설정** → **카카오 로그인** → **동의항목**
   - **카카오톡 서비스 내 친구목록** (선택): **동의 필요** 로 설정
   - **카카오톡 메시지 전송** (선택): **동의 필요** 로 설정
4. **카카오 로그인** → **활성화 설정** ON
5. **플랫폼** → Web: 사이트 도메인에 `https://wherehere-seven.vercel.app` (및 로컬 테스트 시 `http://localhost:3000`) 추가

---

## 2. Supabase에서 카카오 scope 추가

Supabase가 카카오 로그인 시 **친구 목록·메시지** 권한을 요청하도록 해야 합니다.

1. [Supabase Dashboard](https://supabase.com/dashboard) → 프로젝트 선택
2. **Authentication** → **Providers** → **Kakao**
3. **Additional Scopes** (또는 Scopes)에 아래를 추가 (쉼표로 구분):
   ```text
   friends,talk_message
   ```
4. 저장 후, **기존 사용자는 다시 카카오 로그인**해야 새 동의창이 뜹니다.

---

## 3. 앱에서 테스트 순서

1. **로그아웃** 후 **카카오로 로그인**  
   → 동의 화면에 **친구목록·메시지** 항목이 뜨면 **동의** 후 진행
2. **소셜** 탭 이동
3. **카카오 친구에게 초대하기** 버튼 클릭
4. **친구 목록**이 로딩되면 화면 캡처 (아래 4번)
5. 한 명 선택 후 **바로 보내기** 클릭 → 전송 완료되면 캡처 (아래 5번)

---

## 4. 한 장만 제출할 때 (권장)

심사에서 **캡처 1장만** 허용되는 경우, 앱에 **카카오 API 테스트** 전용 화면을 두었습니다.

1. 앱 실행 후 **설정** (⚙️) 진입
2. **카카오 API 테스트 (심사 제출용)** 카드 클릭
3. 화면에 나온 **① 카카오 로그인 → ② 친구목록 동의 → ③ 친구 목록 가져오기 → ④ 친구에게 메시지 전송** 순서대로 진행
   - ① 이미 카카오로 로그인되어 있으면 "완료"로 표시됨
   - ②는 ③에서 친구 목록을 불러오면 동의된 것으로 간주됨
   - ③ **친구 목록 불러오기** 버튼 클릭 → 친구 수·이름이 보이면 완료
   - ④ 목록에서 한 명 골라 **바로 보내기** 클릭 → "완료 — ○○에게 초대 메시지 전송함" 표시
4. **이 테스트 화면 전체**를 한 장 스크린샷으로 캡처
5. 카카오 개발자 콘솔 → 해당 기능 심사 다시 신청 시 **캡처 1장**으로 위 화면 제출

캡처 시 **4단계가 모두 "완료"로 보이고**, 하단 **"캡처 준비 완료"** 문구가 함께 보이도록 찍으면 됩니다.

---

## 5. 심사 제출용 캡처 위치 (4장 제출 시)

아래 **4장**을 순서대로 캡처해 제출하면 됩니다.

| 순서 | 촬영 화면 | 어디서 캡처할지 |
|------|------------|------------------|
| **1** | 카카오 로그인 | 로그인 화면에서 **「카카오로 로그인」** 버튼이 보이는 상태에서 전체 화면 캡처. (버튼이 잘 보이도록) |
| **2** | 친구목록 동의 | 카카오 로그인 후 **동의 화면**이 뜨면, **「카카오톡 서비스 내 친구목록」** 항목이 보이는 상태로 캡처. (동의 후 다음 단계로 넘어가기 전) |
| **3** | 친구 목록 가져오기 | 앱 **소셜** 탭 → **카카오 친구에게 초대하기** 클릭 → **친구 목록이 뜬 모달** 전체 캡처. (친구 닉네임/목록이 보이면 됨) |
| **4** | 친구에게 메시지 전송 | 위 모달에서 **한 명 선택 후 「바로 보내기」** 클릭 → **전송 완료 토스트**(예: "카카오톡으로 초대 메시지를 바로 보냈어요")가 보이는 순간 캡처. 또는 실제 **카카오톡 앱**에서 해당 친구에게 메시지가 도착한 화면 캡처. |

**캡처 요약**

- **1장**: 로그인 화면 (카카오 로그인 버튼)
- **2장**: 동의 화면 (친구목록 동의 항목 노출)
- **3장**: 소셜 탭 → 카카오 친구 초대 → **친구 목록 모달**
- **4장**: **바로 보내기** 후 전송 완료 화면 (또는 카카오톡 수신 화면)

## 6. 문제 해결

- **친구 목록이 비어 있음**  
  카카오톡에서 실제로 친구가 있고, 동의 시 **친구목록** 항목에 동의했는지 확인. Supabase **Additional Scopes**에 `friends`가 들어가 있는지 확인.
- **403 insufficient scopes**  
  사용자가 **친구목록·메시지** 동의를 하지 않은 상태. 로그아웃 후 다시 카카오 로그인해서 동의창에서 두 항목 모두 **동의** 후 진행.
- **토큰이 없다**  
  Supabase에서 Kakao Provider에 `friends,talk_message` scope를 추가한 뒤, **다시 카카오 로그인**해야 `provider_token`에 해당 권한이 포함됩니다.
