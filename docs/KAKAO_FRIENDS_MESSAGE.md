# 카카오톡 친구 목록 / 메시지 API — 심사 통과 체크리스트

카카오 공식 가이드([체크리스트](https://devtalk.kakao.com/t/api-api/116052))에 맞춘 설정·테스트·제출 순서입니다.

**심사에서 요구하는 테스트 이력 4가지**
1. 카카오 로그인
2. '카카오톡 서비스 내 친구목록' 동의항목 동의 받기
3. 친구 목록 가져오기
4. 친구에게 메시지 전송

---

## 사전 필수 (카카오 공식 요약)

- 친구 API·메시지 API는 **제작/운영 중인 앱 내 회원 간 소셜 활동** 목적으로 지원됩니다.
- **자동 메시지는 지양**하며, 검수 시 반려될 수 있습니다.
- 권한 받기 전에는 **팀멤버 추가**하여 테스트 가능 (팀멤버에게 하루 30건까지).
- **친구 목록 조회 조건**:  
  - 앱(사이트)에 **카카오 로그인한 사용자**가 자신의 카카오톡 친구를 조회하고,  
  - **조회될 친구도 앱에 로그인 + 친구목록·메시지 동의**를 해야 목록에 나옵니다.  
  - **동의하지 않으면 카카오톡 친구라도 목록에 조회되지 않습니다.**

---

## 1단계. 팀멤버 추가하여 친구목록 조회 테스트

### 1.1 카카오 개발자 콘솔

1. [Kakao Developers](https://developers.kakao.com) → 내 애플리케이션 → 해당 앱
2. **앱 설정** → **팀멤버** (또는 동일 기능): 친구 목록으로 **조회할 팀멤버 계정** 등록
3. **제품 설정** → **카카오 로그인** → **동의항목**
   - **카카오톡 서비스 내 친구목록**: **이용 중 동의** 또는 **선택 동의** (선택 동의는 권한 신청 후 사용 가능한 경우 있음)
   - **카카오톡 메시지 전송**: 동의 필요 또는 이용 중 동의
4. **제품 설정** → **카카오 로그인** → 활성화 ON
5. **플랫폼** → Web:
   - 사이트 도메인: `https://wherehere-seven.vercel.app` (로컬: `http://localhost:3000`)
   - **Redirect URI** 등록:
     - Supabase: `https://[프로젝트ID].supabase.co/auth/v1/callback`
     - 추가 동의 콜백: `https://wherehere-seven.vercel.app/api/auth/kakao-friends-callback`

### 1.2 내 계정 — 카카오 로그인 + 추가항목 동의

- 앱(WhereHere)에서 **카카오로 로그인**하고, 동의 화면에서 **친구목록·메시지** 항목 **동의**
- (동의창이 안 뜨면) 설정 → **카카오톡 친구 목록 동의하기** 또는, 카카오 API 테스트 화면에서 403 시 **「친구 목록 권한 허용」** 사용

### 1.3 조회될 친구(팀멤버) — 쌍방 동의

- **친구 목록에 나오게 하려면, 그 친구도 앱에 로그인하고 동의해야 합니다.**
- 팀멤버(또는 테스트용 다른 카카오 계정)로 **WhereHere 앱에 카카오 로그인** 후, **친구목록·메시지 동의**까지 진행 (토큰까지 받을 것)
- 이렇게 해야 내 계정으로 “친구 목록 가져오기” 시 해당 팀멤버가 목록에 나타납니다.
- *연결 유지*: 액세스 토큰만 있으면 배치가 연결을 끊을 수 있어, **사용자정보조회 API를 최초 1회 호출**하는 것이 권장됩니다.

### 1.4 앱에서 4단계 테스트

1. **설정** → **카카오 API 테스트 (심사 제출용)** 진입
2. ① 카카오 로그인 완료 확인
3. ② 동의는 ③ 성공 시 완료로 간주
4. ③ **친구 목록 불러오기** 클릭 → 팀멤버(동의 완료한 친구)가 목록에 보이면 성공
5. ④ 목록에서 한 명 선택 후 **바로 보내기** → “완료 — ○○에게 전송함” 확인

## 2단계. 캡처 및 제출 (심사 통과용)

### 한 장으로 4가지 조건 모두 만족시키기 (권장)

제출 화면은 **1장만** 넣어도 됩니다. 아래 순서대로 하면 **한 장**으로 ① 로그인 ② 동의 ③ 친구 목록 조회 ④ 친구에게 메시지 전송이 **모두 보이게** 할 수 있습니다.

1. **팀멤버 1명**을 카카오 콘솔에 등록하고, 그 계정으로도 앱에 로그인·친구목록·메시지 동의까지 해 둡니다.
2. **내 계정**으로 앱 로그인 → 설정 → **카카오 API 테스트 (심사 제출용)** 진입.
3. ③ **친구 목록 불러오기** → 403이면 **「친구 목록 권한 허용」**으로 동의 후 다시 불러오기 → 친구 1명 이상 보이면 OK.
4. ④ 목록에서 한 명 선택 → **바로 보내기** → “완료 — ○○에게 초대 메시지 전송함” 확인.
5. 이때 화면에 **① 완료, ② 완료, ③ 완료(N명 조회됨), ④ 완료(○○에게 전송함), 맨 아래 “캡처 준비 완료”** 가 모두 보이면 **그 상태에서 화면 전체 스크린샷 1장**을 찍습니다.
6. 권한 신청 시:
   - **웹 사이트 또는 앱 마켓 URL**: `https://wherehere-seven.vercel.app` (또는 실제 서비스 URL)
   - **서비스 적용 화면**: 위에서 찍은 **스크린샷 1장** 업로드 (JPG/PNG, 20MB 이내, 개인정보 있으면 마스킹)
   - **신청 사유 및 활용 시나리오** 예시:
     > 로그인 후 [설정] → [카카오 API 테스트(심사 제출용)] 화면에서 (1) 카카오 로그인 (2) 카카오톡 서비스 내 친구목록 동의 (3) 친구 목록 불러오기 (4) 조회된 친구에게 메시지 전송을 순서대로 진행했습니다. 첨부 화면은 4단계까지 완료한 직후 캡처한 것으로, 앱 내 퀘스트/장소를 카카오톡 친구에게 초대하는 기능에 사용합니다.

이렇게 하면 **한 장**으로 “테스트 이력 4가지”를 모두 만족시킨 것으로 심사에 보여줄 수 있습니다.

---

- **여러 장 제출** 시 (참고):
  1. 카카오 로그인 버튼이 보이는 로그인 화면
  2. 동의 화면(친구목록 동의 항목 노출)
  3. 친구 목록이 보이는 화면(설정 → 카카오 API 테스트 또는 소셜 → 카카오 친구 초대)
  4. 친구에게 메시지 전송 완료 화면(또는 카카오톡 수신 화면)

## 3단계. 권한 신청

- **비즈앱 등록** (개인 개발자는 개인 FAQ 참고 후 비즈앱 전환)
- **카카오디벨로퍼스** → **앱** → **추가 기능** → **카카오톡 친구 목록 / 메시지** 권한 신청
- 신청 시 **실제 동작 화면 영상 또는 캡처**로 위 4가지 테스트 이력이 보이도록 제출

## Supabase 설정 (참고)

- Supabase 대시보드: **Authentication** → **Providers** → **Kakao**  
  - **Client ID**(REST API 키), **Client Secret** 입력  
  - **Redirect URLs**에 `https://wherehere-seven.vercel.app` 등 사용 도메인 추가
- Scopes는 Supabase에 입력하는 항목이 없고, 앱에서 로그인 시 `friends`, `talk_message` 를 요청하도록 구현돼 있음.

## 문제 해결

- **친구 목록이 비어 있음**  
  → 조회될 **친구(팀멤버)도 앱에 로그인하고** 친구목록·메시지 **동의**했는지 확인. 동의하지 않으면 목록에 안 나옵니다.
- **403 / 친구 목록 불러오기 실패**  
  → 3단계에서 **「친구 목록 권한 허용」** 버튼으로 카카오 동의창 띄운 뒤 동의하고, 다시 “친구 목록 불러오기” 시도.  
  → 카카오 **플랫폼 키** → 카카오 로그인 리다이렉트 URI에 `https://wherehere-seven.vercel.app/api/auth/kakao-friends-callback` 등록.  
  → Vercel 등 배포 환경에 **KAKAO_CLIENT_SECRET** 환경 변수 설정 (카카오 앱 → 보안 → Client Secret).

- **동의창이 안 뜸**  
  → 시크릿 창에서 시도하거나, 설정에서 **카카오톡 친구 목록 동의하기** 실행 후 재로그인.
- **Unable to exchange external code / auth_exchange_failed (카카오 로그인 실패)**  
  → Redirect URI가 어긋나면 발생합니다. **순서대로** 확인하세요.  
  1. **Supabase** 대시보드 → **Authentication** → **URL Configuration** → **Redirect URLs**에  
     `https://wherehere-seven.vercel.app/auth/callback` 이 **반드시** 포함돼 있는지 확인. (로컬만 쓰면 `http://localhost:3000/auth/callback` 추가.)  
  2. **카카오** 개발자 콘솔 → 내 앱 → **플랫폼** → **Web** → **Redirect URI**에는  
     **Supabase 콜백 주소만** 넣습니다: `https://[프로젝트ID].supabase.co/auth/v1/callback`  
     (프로젝트ID는 Supabase 대시보드 URL에 나옵니다. `https://xxx.supabase.co` 의 xxx 부분.)  
     ※ 친구/메시지 API용 `.../api/auth/kakao-friends-callback` 은 **별도 줄**로 추가하고, 위 Supabase 콜백을 **지우면 안 됨**.)  
  3. 같은 브라우저 창에서 다시 카카오 로그인 시도. (시크릿 창이면 일반 창에서 시도.)
