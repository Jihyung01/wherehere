# 사진 업로드(Storage) 설정

앱에서 **동네 피드·프로필 사진** 업로드가 동작하려면 Supabase Storage에 **`uploads`** 버킷이 있어야 합니다.

## 1. Supabase 대시보드에서 버킷 만들기

1. [Supabase Dashboard](https://app.supabase.com) → 프로젝트 선택
2. 왼쪽 메뉴 **Storage** 클릭
3. **New bucket** 클릭
4. 설정:
   - **Name**: `uploads` (그대로 입력)
   - **Public bucket**: **ON** (체크) — 업로드된 이미지 URL을 피드에서 그대로 쓸 수 있음
   - **File size limit**: `4` MB (선택)
   - **Allowed MIME types**: `image/jpeg`, `image/png`, `image/gif`, `image/webp` (선택)
5. **Create bucket** 클릭

## 2. 정책(Policies) — 꼭 해야 함

버킷을 만들어도 **Policies가 0개**면 "업로드 권한이 없어요"가 납니다. 아래 중 하나로 정책을 추가하세요.

### 방법 A: SQL 한 번 실행 (권장)

1. Supabase 대시보드 → **SQL Editor** → **New query**
2. 프로젝트 루트의 `supabase/migrations/20260305_storage_uploads_policies.sql` 파일 내용을 **전부 복사**해서 붙여넣기
3. **Run** 실행

이렇게 하면 `uploads` 버킷에 대한 업로드(INSERT)·읽기(SELECT)·덮어쓰기(UPDATE) 정책이 생성됩니다.

### 방법 B: 대시보드에서 수동 추가

- Storage > **Policies** 탭 → **New policy** → "For full customization"
- **INSERT** 정책: Target roles `public`, WITH CHECK `bucket_id = 'uploads'`
- **SELECT** 정책: Target roles `public`, USING `bucket_id = 'uploads'`

## 3. 환경 변수 (Vercel 등)

프론트 배포(Vercel)에 다음이 설정되어 있어야 합니다.

- `NEXT_PUBLIC_SUPABASE_URL`: Supabase 프로젝트 URL
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`: Supabase anon public key

설정 후 재배포하면 업로드 API(`/api/upload`)가 해당 버킷을 사용합니다.

## 4. 에러 메시지

- **"사진 저장소가 설정되지 않았어요"**  
  → 위 1~2대로 `uploads` 버킷을 만들고, 3의 환경 변수를 확인하세요.
- **"Bucket not found"**  
  → 버킷 이름이 정확히 `uploads`인지, 해당 Supabase 프로젝트인지 확인하세요.
- **"업로드 권한이 없어요. Storage 버킷 정책을 확인해 주세요."**  
  → **2. 정책(Policies)**에서 방법 A(SQL 실행)를 진행하세요. 버킷은 있는데 정책이 0개일 때 나는 메시지입니다.

---

## 5. "Method Not Allowed" (405) — 동네 피드 글쓰기

**POST** `/api/v1/local/posts`에서 405가 나면 **백엔드(Railway)** 쪽 문제입니다.

- Vercel 환경 변수 **`NEXT_PUBLIC_API_URL`**이 Railway 백엔드 주소가 맞는지 확인 (오타 없이, 예: `https://wherehere-production.up.railway.app`).
- Railway에 **최신 코드**가 배포되어 있는지 확인 (이 레포의 `backend/`에 `routes/local_feed.py`가 포함된 버전이 올라가 있어야 함).
- Railway 서비스가 켜져 있는지, 로그에 에러가 없는지 확인.
