# 사진 업로드(Storage) 설정

앱에서 **동네 피드·프로필 사진** 업로드가 동작하려면 Supabase Storage에 **`uploads`** 버킷이 있어야 합니다.

## 1. Supabase 대시보드에서 버킷 만들기

1. [Supabase Dashboard](https://app.supabase.com) → 프로젝트 선택
2. 왼쪽 메뉴 **Storage** 클릭
3. **New bucket** 클릭
4. 설정:
   - **Name**: `uploads` (그대로 입력)
   - **Public bucket**: **ON** (체크) — 업로드된 이미지 URL을 피드에서 그대로 쓸 수 있음
   - **File size limit**: `4` MB (선택)
   - **Allowed MIME types**: `image/jpeg`, `image/png`, `image/gif`, `image/webp` (선택)
5. **Create bucket** 클릭

## 2. 정책(Policies) 확인

- **Public bucket**으로 만들었으면, 읽기(SELECT)는 이미 허용된 상태입니다.
- 업로드(INSERT)가 안 되면 Storage > **Policies**에서 다음을 추가합니다.
  - **New policy** → "For full customization"
  - Policy name: `Allow uploads to posts`
  - Allowed operation: **INSERT**
  - Target roles: `public` 또는 `authenticated`
  - WITH CHECK expression: `bucket_id = 'uploads'`

## 3. 환경 변수 (Vercel 등)

프론트 배포(Vercel)에 다음이 설정되어 있어야 합니다.

- `NEXT_PUBLIC_SUPABASE_URL`: Supabase 프로젝트 URL
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`: Supabase anon public key

설정 후 재배포하면 업로드 API(`/api/upload`)가 해당 버킷을 사용합니다.

## 4. 에러 메시지

- **"사진 저장소가 설정되지 않았어요"**  
  → 위 1~2대로 `uploads` 버킷을 만들고, 3의 환경 변수를 확인하세요.
- **"Bucket not found"**  
  → 버킷 이름이 정확히 `uploads`인지, 해당 Supabase 프로젝트인지 확인하세요.
