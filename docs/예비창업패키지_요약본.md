# WhereHere 예비창업패키지 지원서 (요약본)

## 📌 한 줄 요약
**"AI가 내 기분에 맞는 장소를 퀘스트로 추천하고, 게임처럼 동네를 정복하는 소셜 탐험 플랫폼"**

---

## 🎯 핵심 내용 (5분 읽기)

### 1. 해결하는 문제
❌ **기존 지도 앱의 한계**
- "오늘 뭐하지?" → 12분 검색, 3.5개 앱 전환
- 모두에게 같은 "핫플" 추천 → 개인화 없음
- 익숙한 장소만 반복 (평균 활동 반경 2.3km)

✅ **WhereHere의 해결책**
- AI가 3곳 퀘스트 추천 → 선택만 하면 됨
- 역할(5가지) + 기분(10가지) = 50가지 조합 개인화
- 게임처럼 레벨업 → 동네 탐험 동기 부여

### 2. 핵심 기능 (3가지)

#### ① AI 초개인화 추천
```
역할 선택 (탐험가/치유자/수집가/연결자/달성자)
  +
기분 선택 (호기심/평온/활기/우울/...)
  +
현재 위치 (GPS)
  ↓
AI가 딱 맞는 장소 3곳 추천
```

#### ② 위치 기반 게이미피케이션
- **레벨 & XP**: 50레벨, 방문할수록 성장
- **스트릭**: 연속 방문 보너스 (최대 +100% XP)
- **챌린지**: 18개 (일일/주간/업적/소셜/탐험)
- **동네 정복 지도**: 헥사곤으로 시각화

#### ③ 소셜 커뮤니티
- **협동 퀘스트**: 친구와 함께 도전 (최대 4명)
- **카카오 친구 찾기**: 카카오톡 친구 자동 매칭
- **동네 피드**: 좋아요/댓글/리뷰 (실시간 알림)

### 3. 차별점

| 구분 | 네이버/카카오맵 | WhereHere |
|------|----------------|-----------|
| 추천 방식 | 검색 중심 | **AI 퀘스트** |
| 개인화 | 검색 기록만 | **역할+기분+AI** |
| 동기 부여 | 없음 | **레벨/XP/지도** |
| 위치 인증 | 없음 | **100m 검증** |
| 소셜 | 공유만 | **협동 퀘스트** |

### 4. 기술 스택
- **Frontend**: Next.js 14, TypeScript, React 18
- **Backend**: FastAPI, Python 3.10, PostgreSQL + PostGIS
- **AI**: Claude 3.5 Sonnet (서사 생성)
- **Infrastructure**: Supabase, Vercel, Railway

### 5. 현재 상태
- ✅ **개발 완료도**: 85%
- ✅ **베타 테스트**: 50명 (만족도 92%)
- ✅ **실데이터**: 3,000+ 장소 (서울·수도권)
- ✅ **카카오 승인**: 친구/메시지 API
- ✅ **배포 완료**: Vercel (프론트엔드)

## 🎯 성공 요인

### 1. 검증된 아이템
- 베타 테스터 주간 방문 **78% 증가** (3.2회 → 5.7회)
- "새로운 장소 발견이 재미있다" **92%**

### 2. 명확한 차별점
- AI 추천 + 위치 검증 + 게이미피케이션 **3박자**
- 경쟁사 모방 어려움 (특허 출원 예정)

### 3. 바이럴 설계
- 협동 퀘스트 → 친구 초대 필수
- 카카오톡 공유 → 자연스러운 확산
- 목표 바이럴 계수: **1.5** (1명이 1.5명 초대)

### 4. 성장하는 시장
- 글로벌 LBS 시장: 연평균 **+24.3%** 성장
- 국내 로컬 커머스: **32조원** (2025)
- MZ세대 경험 소비 증가
