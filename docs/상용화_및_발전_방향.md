# WhereHere – 현황 분석 및 상용화·발전 방향

> **목표**: 지금 상태를 기준으로, 더 매력적이고 상용화 가능한 앱이 되기 위한 현황 분석·방향성·기능·UI 제안

---

## 1. 현황 분석

### 1.1 강점 (이미 갖춘 것)

| 영역 | 내용 |
|------|------|
| **핵심 루프** | 역할·무드 → 퀘스트 추천 → 체크인·리뷰 → XP → 나의 지도. 한 사이클이 완성됨 |
| **위치 기반** | 100m 도착 인정, 체크인 위치 검증, 앱 내 경로 지도. “실제로 간다”가 검증됨 |
| **데이터** | 실데이터(places 3000+), Supabase, 방문 기록·추천 연동 |
| **AI 차별점** | Claude 서사, 역할/무드 기반 추천. 단순 “근처 맛집”이 아님 |
| **인증** | 카카오/구글/이메일. 국내 사용자 진입 장벽 낮음 |

### 1.2 상용화 관점에서의 갭

| 갭 | 설명 | 영향도 |
|----|------|--------|
| **레벨/XP가 실제 데이터와 미연동** | `/me/stats`가 목(mock) 데이터. 방문 기반 레벨·스트릭이 아님 | 사용자 몰입·리텐션 |
| **소셜·공유가 약함** | “친구에게 공유”는 있으나, 친구 목록·팔로우·함께하기 없음 | 바이럴·재방문 |
| **알림 없음** | 푸시/실시간 알림 없음. 재방문 유도가 약함 | 리텐션 |
| **콘텐츠 생성이 1방향** | 사용자가 장소를 등록·수정하는 크리에이터 모드 없음 | UGC·데이터 확장 |
| **수익화 없음** | 광고·구독·제휴 등 없음 | 상용화 지속 가능성 |
| **온보딩·설명이 짧음** | 첫 진입 시 “왜 쓰는 앱인지” 한 번에 와닿기 어려울 수 있음 | 전환율 |

### 1.3 정리

- **프로덕트 핵심(퀘스트 → 체크인 → XP → 지도)** 은 갖춰져 있음.
- **“계속 쓰게 만드는 요소”(레벨/스트릭 실데이터, 알림, 소셜)** 와 **“앱이 커지게 만드는 요소”(UGC, 수익화)** 가 다음 단계.

## 2. 앞으로의 방향성 (제안)

### 2.1 포지셔닝

- **한 줄**: “내 기분과 역할에 맞는 장소를 퀘스트로 받고, 가서 인증하면 XP와 나만의 지도가 쌓이는 **도심 탐험 앱**”
- **차별점**:  
  - 단순 추천이 아니라 **퀘스트(미션)**  
  - **위치 인증(100m)** 으로 “진짜 갔다” 검증  
  - **나의 지도**로 탐험 이력이 시각화  
  - **역할·무드·AI 서사**로 개인화

### 2.2 단계별 목표

| 단계 | 목표 | 기간 감 |
|------|------|--------|
| **안정화** | 레벨/XP/스트릭 실데이터 연동, 알려진 이슈 제로 | 1~2주 |
| **몰입** | 알림(푸시/실시간), 온보딩 개선, 레벨 보상 체감 | 2~4주 |
| **확장** | 소셜(친구·팔로우·공유 강화), 크리에이터 모드(장소 등록) | 1~2개월 |
| **상용화** | 수익화(선택: 구독/제휴/광고), PWA·앱처럼 쓰기 | 2개월+ |

## 3. 발전시킬 기능 (우선순위)

### 3.1 단기 (안정화·몰입)

| 순위 | 기능 | 설명 | 기대 효과 |
|------|------|------|------------|
| 1 | **레벨/XP/스트릭 실데이터** | visits 기반으로 total_xp, level, current_streak, xp_to_next_level 계산 후 `/me/stats` 또는 새 API에서 반환. 프로필 UI는 이미 있음 | “진짜 성장” 체감, 리텐션 |
| 2 | **온보딩 플로우** | 첫 로그인 시 2~3장: “역할 고르기 → 퀘스트 받기 → 도착 인정” 한 번 요약. 스킵 가능 | 전환율, 이해도 |
| 3 | **푸시/알림 (선택)** | “오늘 퀘스트 추천”, “N일 연속 달성”, “친구가 새 장소 체크인” 등. Supabase 또는 FCM | 재방문, 몰입 |

### 3.2 중기 (확장)

| 순위 | 기능 | 설명 | 기대 효과 |
|------|------|------|------------|
| 4 | **소셜: 친구·팔로우** | 친구 추가, 팔로우 목록, “친구의 최근 방문” 노출(선택). DB에 `friends`/`follows` 테이블 | 바이럴, 재방문 |
| 5 | **크리에이터 모드** | 사용자가 “장소 제안” 또는 “등록”(이름, 주소, 카테고리, 사진). 검수 후 places 반영 | UGC, 장소 데이터 확장 |
| 6 | **실시간 알림** | Supabase Realtime으로 “친구 체크인”, “퀘스트 리마인더” 등 | 몰입, 소셜 느낌 |

### 3.3 장기 (상용화·다양성)

| 순위 | 기능 | 설명 | 기대 효과 |
|------|------|------|------------|
| 7 | **수익화** | 프리미엄 퀘스트, “이번 달 추천 무제한”, 제휴 할인 등 (구독 또는 제휴) | 지속 가능한 운영 |
| 8 | **모바일 앱 (React Native)** | 동일 로직으로 iOS/Android 앱. 푸시·앱스토어 노출 | 접근성, 신뢰 |
| 9 | **관리자 대시보드** | 장소/방문/사용자 현황, UGC 검수, 콘텐츠 관리 | 운영 효율 |
| 10 | **다국어(i18n)** | 영어 등. 해외 사용자 또는 관광객 타깃 | 확장 |

## 4. UI·UX 개선 제안

### 4.1 첫인상·온보딩

- **랜딩/첫 화면**: “오늘 기분과 역할을 골라보세요” + 역할/무드 카드가 한 눈에 보이게. 현재도 있으나, **한 문장 슬로건**과 **예시 퀘스트 한 줄**이 있으면 좋음.
- **첫 로그인**: 2~3단계 온보딩(역할 소개 → 퀘스트 받기 → 도착 인정/지도 소개). “다음” / “건너뛰기” 제공.

### 4.2 퀘스트·추천 화면

- **퀘스트 카드**: 거리·카테고리·점수 외에 **서사 한 줄** 미리보기, **“경로 보기”** 터치 시 앱 내 지도로 바로 이동(이미 구현된 경로 지도와 연결).
- **빈 상태**: 추천 0건일 때 “조금 더 넓은 범위로 찾아볼까요?” + 반경 넓히기 또는 역할/무드 변경 유도.

### 4.3 수락한 퀘스트·경로

- **경로 지도**: 이미 앱 내 지도에 경로 표시됨. **“현재 위치 새로고침”** 버튼이 있으면 이동 중에 유용.
- **도착 인정**: “약 Nm 남음” 문구와 **100m 이내일 때만 활성화되는 버튼** 상태가 명확하면 좋음(이미 구현).

### 4.4 나의 지도

- **지도 탭**: 기간 필터 + “이번 달 탐험 반경” 등 한 줄 인사이트. 이미 통계/스타일 탭이 있으므로, **탭 전환만으로도 스토리텔링**이 느껴지게.
- **방문 카드**: 장소 사진(있으면), 카테고리 뱃지, XP, 한 줄 리뷰 요약.

### 4.5 프로필·레벨

- **레벨 진행바**: 이미 있음. 실데이터 연동 시 “다음 레벨까지 N XP”가 실제 방문으로 변하는 걸 강조.
- **업적/뱃지**: “첫 방문”, “7일 연속”, “10곳 방문” 등 3~5개 뱃지 아이콘. 나중에 백엔드와 연동.

### 4.6 설정·전체 톤

- **다크모드**: 유지. **시스템 설정 따라가기** 옵션 있으면 좋음.
- **일관된 색/타이포**: 주 색상(주황 #E8740C)과 카드·버튼 스타일을 화면마다 통일.
- **로딩·에러**: “추천 받는 중…”, “위치 확인 중…” 등 상태 메시지와 재시도 버튼 명확히.
