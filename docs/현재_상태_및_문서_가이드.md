## 1. 현재 구현된 기능 (동작 중)

### 인증·계정
| 기능 | 설명 |
|------|------|
| 카카오 로그인 | OAuth, 콜백 `/auth/callback` → 세션 저장 |
| 구글 로그인 | 동일 |
| 이메일 회원가입/로그인 | Supabase (이메일 인증 필요 시 설정) |
| 로그아웃 | 설정 > 개인정보에서 로그아웃 |
| 사용자별 데이터 분리 | 로그인 시 `user.id` 기준으로 방문·추천·나의 지도 분리 |

### 메인 앱 (CompleteApp)
| 기능 | 설명 |
|------|------|
| 역할 선택 | 탐험가/힐러/예술가/미식가/도전자 |
| 무드 선택 | 호기심/지침/영감/배고픔/모험 |
| 퀘스트 추천 | 위치·역할·무드 기반 추천 받기 |
| 퀘스트 수락 → 체크인 → 리뷰 | 방문 기록 생성, XP 반영 |
| 나의 지도 이동 | `/my-map-real` 링크 |
| 설정 | 알림/위치/개인정보/도움말, 다크모드, 로그인 상태·닉네임 표시 |

### 나의 지도 (`/my-map-real`)
| 기능 | 설명 |
|------|------|
| 기간 필터 | 오늘/이번 주/이번 달/전체 |
| 지도 탭 | 카카오맵, 내 방문 마커+경로선 (주황색) |
| 통계 탭 | 방문 수, 카테고리 분포, AI 인사이트 |
| 스타일 탭 | 스타일 분석, 가치 재분배 팁 |
| 최근 방문 목록 | 방문 일시, XP 등 |
| **카테고리/스타일** | 방문의 카테고리는 **places** 테이블의 `primary_category`에서 가져옴. places가 비어 있으면 "기타"로 표시되고, 스타일은 "초보 탐험가" 등으로 표시됨. places 수집 스크립트로 장소를 채우면 실제 카테고리·스타일이 반영됨. |

### 백엔드 API (실제 구현됨)
| 경로 | 용도 |
|------|------|
| `GET/POST /api/v1/visits` | 방문 조회·생성 (user_id 기준) |
| `POST /api/v1/ai/pattern/analyze` | 패턴 분석 |
| `GET /api/v1/ai/personality/{user_id}` | 성격/스타일 |
| 추천·퀘스트·챌린지·소셜 등 | `backend/routes/` 참고 |

### 배포
| 항목 | 설명 |
|------|------|
| 프론트 | Vercel (예: wherehere-seven.vercel.app) |
| 백엔드 | 별도 호스트 시 `NEXT_PUBLIC_API_URL` 설정 |
| Supabase | Auth + DB (실데이터 시 스키마/시드 적용) |
