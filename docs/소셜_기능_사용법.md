# 소셜 기능 사용법

> WhereHere 앱의 **소셜 탭**(피드·팔로우)이 어떻게 동작하는지, 개발자/운영자가 어떻게 활용할 수 있는지 정리한 문서입니다.

---

## 1. 소셜 기능이란?

- **피드**: “나 + 내가 팔로우한 사람”의 **최근 활동**을 시간순으로 보는 공간 (당근 동네생활·오픈채팅 피드 느낌).
- **팔로우**: 다른 사용자를 “팔로우”하면, 그 사람의 체크인·활동이 **내 피드**에 보입니다.

---

## 2. 앱에서 할 수 있는 것 (현재)

| 동작 | 설명 |
|------|------|
| **소셜 탭 보기** | 하단 네비에서 **소셜** 탭 선택 → “동네 탐험가들의 체크인과 소식” 피드 진입. |
| **피드 내용** | **나** + **내가 팔로우한 사용자**의 최근 활동만 노출. 활동 예: “OO님이 [장소명]에서 체크인, +N XP”, 날짜. |
| **빈 피드** | 팔로우한 사람이 없거나, 나·팔로우한 사람 모두 최근 활동이 없으면 “퀘스트를 완료하거나 친구를 팔로우하면 여기에 체크인 소식이 쌓여요” 문구 표시. |

**현재 앱에는 “팔로우하기” 버튼·사용자 검색 UI가 없습니다.**  
팔로우/언팔로우는 **API로만** 가능합니다 (아래 4절).

---

## 3. 피드가 채워지는 방식

1. **체크인 시 자동 생성**  
   사용자가 퀘스트를 완료하고 **체크인**하면, 백엔드에서 `feed_activities` 테이블에 활동 한 건이 저장됩니다.  
   - 저장 주소: `backend/routes/visits.py` 체크인 성공 처리 시 `create_feed_activity()` 호출.
2. **피드 조회**  
   - 소셜 탭에서 `GET /api/v1/social/feed?user_id=현재사용자ID` 호출.  
   - 백엔드가 “내가 팔로우한 사람 ID 목록”을 구한 뒤, “나 + 팔로우한 사람”의 최근 `feed_activities`를 모아서 반환.  
   - 프론트는 이 목록을 시간순으로 보여줍니다.

정리하면: **퀘스트 완료(체크인) → 피드 활동 1건 생성 → 소셜 탭에서 “나 + 팔로우한 사람” 피드에만 노출.**

---

## 4. API로 팔로우/언팔로우·피드 보기

DB와 마이그레이션이 적용된 상태에서 아래 API를 사용할 수 있습니다.

### 4.1 피드 가져오기

```http
GET /api/v1/social/feed?user_id={현재_사용자_ID}&limit=50
```

- **응답**: `{ "activities": [ ... ], "following_ids": [ ... ] }`  
- `activities`: 나 + 팔로우한 사람의 최근 활동 목록 (최신순).  
- `following_ids`: 내가 팔로우 중인 사용자 ID 목록.

### 4.2 팔로우하기

```http
POST /api/v1/social/follow?follower_id={내_ID}&following_id={팔로우할_사용자_ID}
```

- **응답**: `{ "success": true }` 등.  
- 이후부터는 “팔로우한 사용자”의 활동이 내 피드에 포함됩니다.

### 4.3 팔로우 해제

```http
DELETE /api/v1/social/follow?follower_id={내_ID}&following_id={언팔로우할_사용자_ID}
```

### 4.4 내가 팔로우하는 목록

```http
GET /api/v1/social/following?user_id={내_ID}
```

- **응답**: `{ "following_ids": [ "uuid1", "uuid2", ... ] }`

## 5. DB 테이블 (참고)

- **follows**: `follower_id`, `following_id` — 누가 누구를 팔로우하는지.
- **feed_activities**: `user_id`, `type`, `place_id`, `place_name`, `xp_earned`, `content`, `created_at` — 체크인 등으로 생성된 활동.

마이그레이션 파일: `supabase/migrations/20260227_notifications_social.sql`  
(이 파일을 Supabase SQL Editor에서 실행해 두어야 피드·팔로우 API가 동작합니다.)

## 6. 이후 확장 시 참고

- **앱 내 팔로우 UI**: “친구 찾기” / “사용자 검색” → 프로필에서 “팔로우” 버튼 → 위 `POST /social/follow` 호출.  
- **알림**: “팔로우한 OO님이 새 장소에 체크인했어요” 등은 `notifications` 테이블 + 푸시 연동으로 확장 가능.
