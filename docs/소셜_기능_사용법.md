# 소셜 기능 사용법

> WhereHere 앱의 **소셜 탭**(피드·팔로우)이 어떻게 동작하는지, 개발자/운영자가 어떻게 활용할 수 있는지 정리한 문서입니다.

---

## 1. 소셜 기능이란?

- **피드**: “나 + 내가 팔로우한 사람”의 **최근 활동**을 시간순으로 보는 공간 (당근 동네생활·오픈채팅 피드 느낌).
- **팔로우**: 다른 사용자를 “팔로우”하면, 그 사람의 체크인·활동이 **내 피드**에 보입니다.

---

## 2. 앱에서 할 수 있는 것 (현재)

소셜 탭은 **동네** / **친구** 두 가지로 나뉩니다.

### 동네 탭 (로컬 피드)

| 동작 | 설명 |
|------|------|
| **홈** | 동네 요약 지표(게시글 수, 열린 모임, 리뷰 평균 등). |
| **작성** | 동네 이야기 / 리뷰(별점) / 모임(시간) 게시글 작성. 제목, 내용, 선택 사항(이미지 URL, 장소명). |
| **피드** | 동네 게시글 목록(최신순). 각 글에 댓글 작성·표시, **카카오 공유**, **인스타 카드**(스토리 크기 이미지 생성 후 공유/다운로드 + 캡션 복사). |

### 친구 탭 (기존 체크인 피드)

| 동작 | 설명 |
|------|------|
| **내 친구 코드** | 복사 후 카카오톡/인스타 DM으로 친구 초대. |
| **친구 찾기** | 닉네임·아이디 검색 → **팔로우** / **채팅** 버튼. |
| **피드** | **나** + **내가 팔로우한 사용자**의 체크인 활동만 노출. (“OO님이 [장소명]에서 체크인, +N XP”) |

---

## 3. 피드가 채워지는 방식

1. **체크인 시 자동 생성**  
   사용자가 퀘스트를 완료하고 **체크인**하면, 백엔드에서 `feed_activities` 테이블에 활동 한 건이 저장됩니다.  
   - 저장 주소: `backend/routes/visits.py` 체크인 성공 처리 시 `create_feed_activity()` 호출.
2. **피드 조회**  
   - 소셜 탭에서 `GET /api/v1/social/feed?user_id=현재사용자ID` 호출.  
   - 백엔드가 “내가 팔로우한 사람 ID 목록”을 구한 뒤, “나 + 팔로우한 사람”의 최근 `feed_activities`를 모아서 반환.  
   - 프론트는 이 목록을 시간순으로 보여줍니다.

정리하면: **퀘스트 완료(체크인) → 피드 활동 1건 생성 → 소셜 탭에서 “나 + 팔로우한 사람” 피드에만 노출.**

---

## 4. API로 팔로우/언팔로우·피드 보기

DB와 마이그레이션이 적용된 상태에서 아래 API를 사용할 수 있습니다.

### 4.1 피드 가져오기

```http
GET /api/v1/social/feed?user_id={현재_사용자_ID}&limit=50
```

- **응답**: `{ "activities": [ ... ], "following_ids": [ ... ] }`  
- `activities`: 나 + 팔로우한 사람의 최근 활동 목록 (최신순).  
- `following_ids`: 내가 팔로우 중인 사용자 ID 목록.

### 4.2 팔로우하기

```http
POST /api/v1/social/follow?follower_id={내_ID}&following_id={팔로우할_사용자_ID}
```

- **응답**: `{ "success": true }` 등.  
- 이후부터는 “팔로우한 사용자”의 활동이 내 피드에 포함됩니다.

### 4.3 팔로우 해제

```http
DELETE /api/v1/social/follow?follower_id={내_ID}&following_id={언팔로우할_사용자_ID}
```

### 4.4 내가 팔로우하는 목록

```http
GET /api/v1/social/following?user_id={내_ID}
```

- **응답**: `{ "following_ids": [ "uuid1", "uuid2", ... ] }`

---

## 5. DB 테이블 (참고)

- **follows**: `follower_id`, `following_id` — 누가 누구를 팔로우하는지.
- **feed_activities**: `user_id`, `type`, `place_id`, `place_name`, `xp_earned`, `content`, `created_at` — 체크인 등으로 생성된 활동.
- **local_posts**: 동네 게시글 (`type`: story/review/gathering, `title`, `body`, `rating`, `meet_time`, `image_url`, `place_name`, `area_name`, `author_id`).
- **local_comments**: 게시글 댓글 (`post_id`, `author_id`, `body`).

마이그레이션: `supabase/migrations/20260227_notifications_social.sql`, `supabase/migrations/20260303_local_posts_comments.sql`  
(SQL Editor에서 실행해 두어야 피드·팔로우·동네 게시글/댓글이 동작합니다.)

---

## 6. 로컬 피드 API (동네 게시글·댓글)

- `GET /api/v1/local/posts?scope=neighborhood&area_name=...&limit=50` — 동네 게시글 목록.
- `GET /api/v1/local/posts?scope=following&user_id=...` — 내 + 팔로우한 사람 게시글.
- `POST /api/v1/local/posts` — 게시글 작성 (body: author_id, type, title, body, rating, meet_time, image_url, place_name, place_address, area_name).
- `GET /api/v1/local/posts/{post_id}/comments` — 댓글 목록.
- `POST /api/v1/local/posts/{post_id}/comments` — 댓글 작성 (body: author_id, body).

## 7. 이후 확장 시 참고

- **동네 지표**: 댓글 수 등은 별도 집계 API로 확장 가능.
- **알림**: “팔로우한 OO님이 새 장소에 체크인했어요” 등은 `notifications` 테이블 + 푸시 연동으로 확장 가능.

## 8. 카카오톡/인스타 친구 연동 가능 여부 (MZ 타깃 확장)

전화번호·카카오톡 친구·인스타 팔로우를 연동해 “친구 추가”를 더 쉽게 할 수 있는지 정리했습니다.

### 8.1 카카오톡 친구 목록

| 항목 | 내용 |
|------|------|
| **가능 여부** | **제한적으로 가능** (목록 조회·선택 UI만) |
| **API** | 카카오톡 소셜 API `GET /v1/api/talk/friends` (앱 권한·동의 항목 `friends` 필요) |
| **제한** | 카카오 정책상 **조회한 친구 목록을 DB 등 외부에 저장·보관하는 것은 금지**됩니다. “메시지 발송 대상 선택” 등 목적으로 필요할 때마다 **실시간 조회만** 가능. |
| **실현 가능한 UX** | (1) “카카오 친구에서 초대하기” 버튼 → 친구 목록 API 호출 → 화면에 목록 표시 → 사용자가 선택 시 **카카오톡 메시지/공유로 앱 초대 링크 전송**. (2) “친구가 WhereHere에 가입했는지”를 우리 DB에 저장하지 않고, “이 사람에게 초대 보내기”만 가능. 즉 **초대 플로우**에는 활용 가능, **팔로우 자동 동기화**는 정책상 불가. |

### 8.2 전화번호 동기화

| 항목 | 내용 |
|------|------|
| **가능 여부** | **카카오 API만으로는 불가** |
| **이유** | 카카오 API는 “카카오톡 친구 목록”만 제공하며, 전화번호부 연동/동기화 API는 제공하지 않음. |
| **대안** | 주소록 기반 “친구 찾기”를 쓰려면 별도 전화번호 권한 + 서버에서 해시 매칭 등 개인정보 처리 필요하며, 정책·개인정보보호법 부담이 큼. 현실적으로는 **친구 코드·닉네임 검색** 또는 **카카오 친구 초대** 쪽이 부담이 적음. |

### 8.3 인스타그램 팔로우 연동

| 항목 | 내용 |
|------|------|
| **가능 여부** | **일반 앱에서는 사실상 불가** |
| **이유** | Meta의 **Instagram Graph API**는 비즈니스/크리에이터 계정용이며, 일반 사용자의 **팔로워/팔로잉 목록을 반환하는 공식 API는 제공하지 않음**. 서드파티 스크래핑/비공식 API는 이용약관 위반·계정 제재 위험이 있음. |
| **실현 가능한 UX** | 인스타 **공유**는 현재처럼 “인스타 카드(스토리/피드 크기 이미지) + 캡션 복사” 또는 Web Share로 처리하는 방식이 현실적. “인스타 아이디로 연동해 팔로우 동기화”는 **공식 API로는 불가**라 제외하는 편이 안전함. |

### 8.4 정리 (MZ·상용화 관점)

- **추천 (부담 적고 실현 가능)**  
  - **친구 코드 + 닉네임/아이디 검색** (현재 방식) 유지.  
  - **“카카오 친구에게 초대하기”** 추가: 카카오 친구 목록 API로 목록만 표시 → 선택 시 앱 가입 링크를 카카오톡으로 보내기. (친구 목록 자체는 저장하지 않음.)  
  - **인스타**는 “공유하기(카드 이미지 + 캡션)”만 강화.

- **비추천 (정책·리스크)**  
  - 카카오 친구 목록을 우리 DB에 저장해 “카카오 친구 = 자동 팔로우”처럼 쓰는 것.  
  - 인스타 팔로워/팔로잉 목록 연동 또는 스크래핑.

요약하면, **전화번호·인스타 팔로우 “동기화”는 무리**가 있고, **카카오 친구는 “초대하기” 용도로만** 쓰는 확장이 세련되면서도 정책에 맞는 방향입니다.
