# 장소 추천 & 할일 UX — 심플·직관·매력 개선 방향

현재 WhereHere 구조를 기준으로, **장소 추천**과 **할일(미션)**을 더 심플하고 직관적으로 주고, 사용자가 “하고 싶다”고 느끼게 만드는 방향을 정리했습니다.

---

## 1. 현재 상태 요약

| 구분 | 현재 흐름 | 사용자 입장에서의 느낌 |
|------|-----------|------------------------|
| **장소 추천** | 홈 → "오늘의 한 곳" 1장 카드 / 또는 "기분 맞춤 탐험" → 역할 → 기분 → 퀘스트 3곳 | 한 곳은 바로 보이지만 **이유·스토리가 약함**. 기분 맞춤은 **3단계(역할→기분→목록)**라 진입 부담 |
| **할일(미션)** | 퀘스트 수락(accepted) 후에만 미션 3~4개 노출 (도착 인증, 별점, 사진, 텍스트 등) | **장소 가기 전에는 할일이 안 보임** → "뭘 하면 되는지"가 나중에만 보임 |
| **진입점** | 홈에 "오늘의 한 곳" / "기분 맞춤 탐험" / "동네 정복 지도" / "동네 피드"가 나란히 있음 | 선택지가 많아 **첫 행동이 분산**될 수 있음 |

---

## 2. “하고 싶다”로 만드는 원칙

- **한 눈에 끝이 보이기**: “이거 하나만 하면 된다”가 바로 보이게
- **할일은 미리 보이기**: 장소 가기 전에 “가면 이거 하면 돼”가 보여서 기대감·동기 부여
- **클릭/탭 수 줄이기**: 추천까지 1~2탭, 할일까지 0~1탭
- **말로 꽂히게**: 점수·거리보다 “왜 오늘 이곳인지” 한 문장 스토리

---

## 3. 장소 추천 — 심플·직관·매력

### 3.1 홈: “오늘의 한 곳”을 **한 문장 스토리**로

- **현재**: 이름, 주소, 점수, 거리, reason 텍스트
- **개선**:
  - **헤드라인 한 문장** 예: “지친 오후, 조용히 쉬기 딱 좋은 동네 카페”
  - 카드 상단에 **대표 이미지** 1장 (있으면) → 터치 욕구 up
  - CTA는 하나로 통일: **“이곳 가기”** (경로 보기로 바로 연결)
  - 부가 정보(거리, 점수)는 작게 하단 또는 접을 수 있게

→ **결과**: “오늘 뭐 하지?” → 앱 열면 **한 문장 + 한 장소 + 한 버튼**으로 결정 부담 감소.

### 3.2 “기분 맞춤 탐험” 경로 단순화

- **현재**: 역할 선택 → 기분 선택 → 퀘스트 3곳 (3단계)
- **개선 옵션**:
  - **A) 1탭 축약**: “지금 기분”만 고르면 역할은 백엔드 기본값/자동 → 곧바로 퀘스트 3곳
  - **B) 홈에서 바로 퀘스트**: “기분 맞춤” 탭을 “지금 기분 한 번만 골라줘” → 다음 화면이 곧 3곳 (역할은 이전 선택 기억 또는 기본값)
  - **C) 역할+기분 한 화면**: 스크롤 한 번에 역할 5개 + 기분 5개 → 고르면 다음이 퀘스트 목록

→ **목표**: **최대 2탭**(기분 선택 → 장소 목록)으로 “맞춤 추천”이 나오게.

### 3.3 퀘스트 카드 — “하고 싶다” 한 줄

- **현재**: 이름, 카테고리, 점수, 주소, 거리, reason
- **개선**:
  - **첫 줄**: 역할/기분에 맞는 **한 줄 이유** (예: “피곤한 오늘, 조용히 차 한 잔하기 좋은 곳”)
  - 그 다음에 장소명·거리
  - **할일 미리보기** (아래 4절): “이곳 가면 → 도착 인증 + 사진 1장 + 한 줄 리뷰”처럼 3개 이내로 요약

→ **결과**: “이 장소 가면 뭘 하면 되는지”가 **고르는 순간**부터 보임.

---

## 4. 할일(미션) — 심플·직관·미리 보기

### 4.1 퀘스트 수락 전에 “할일 미리보기”

- **현재**: 퀘스트 수락(accepted) 후에만 미션 3~4개 표시
- **개선**:
  - **퀘스트 카드** 또는 **“오늘의 한 곳” 카드**에 “이곳에서 할 일” 3개 이내 요약
    - 예: `📍 도착 인증 · 📸 베스트 앵글 · ✍️ 한 줄 리뷰` (아이콘 + 짧은 라벨)
  - 퀘스트 상세(accepted) 화면에서만 전체 미션 상세(설명, 입력 필드) 노출

→ **결과**: “가기 전에 뭘 하면 되는지”가 보여서 **기대감**과 **동기** 형성.

### 4.2 할일 개수·난이도 심플하게

- **현재**: 도착 인증(필수) + 별점(필수) + 역할/기분 기반 선택 미션 2~3개
- **유지하면서 보여주는 방식**:
  - **필수 2개만** 강조: “도착하기 + 별점” → “이 두 가지만 하면 완료”
  - 나머지는 “보너스 미션(추가 XP)”로 표시 → 부담 감소, 선택 느낌
  - 리스트는 **체크리스트 형태**로, 완료 시 체크 표시로 성취감

### 4.3 “오늘의 한 곳”에 할일 요약 붙이기

- 홈 “오늘의 한 곳” 카드 하단에:
  - **“이곳 가면”**: `도착 인증 · 별점 · (선택) 사진 1장` 처럼 3개 이하로 짧게
  - “자세한 미션은 경로 보기 후 확인” 링크 하나로 accepted 화면 연결

→ **결과**: 홈에서도 **장소 + 할일**이 한 블록으로 인지됨.

---

## 5. 진입점 정리 — 한 가지 행동 우선

- **현재**: 홈에 [동네 정복 지도] [기분 맞춤 탐험] [동네 피드]가 비슷한 비중
- **개선**:
  - **첫 번째 행동**을 하나로: “오늘의 한 곳” (한 문장 스토리 + 이미지 + “이곳 가기”)
  - 그 다음 블록으로 “다른 추천 받기” → 기분 1탭으로 퀘스트 3곳
  - “동네 지도”·“동네 피드”는 그 다음 또는 하단 네비로

→ **결과**: “앱 열면 → 오늘 갈 곳 하나 + 할일 3개”가 3초 안에 보이게.

---

## 6. 구현 우선순위 제안

| 순위 | 항목 | 기대 효과 | 난이도 |
|------|------|-----------|--------|
| 1 | “오늘의 한 곳” 카드에 **한 문장 스토리(헤드라인)** + **할일 3개 요약** | 직관·동기 up | 낮음 |
| 2 | 퀘스트 카드에 **할일 미리보기** 3개 (아이콘+라벨) | “하고 싶다” up | 낮음 |
| 3 | 기분 맞춤 **1탭 축약** (기분만 선택 → 퀘스트) 또는 역할+기분 한 화면 | 진입 부담 감소 | 중간 |
| 4 | accepted 화면에서 **필수 2개 vs 보너스 미션** 시각적 구분 | 심플함·성취감 | 낮음 |
| 5 | “오늘의 한 곳” **대표 이미지** 노출 (있을 때) | 매력도 up | 낮음 |

---

## 7. 한 줄 요약

- **장소 추천**: “오늘의 한 곳”을 **한 문장 스토리 + 한 CTA**로, 기분 맞춤은 **1~2탭**으로 끝나게.
- **할일**: **퀘스트/오늘의 한 곳 카드에서 미리** “이곳 가면 이 3개”를 보여주고, 상세는 수락 후에만.
- **진입점**: 홈 첫 블록은 **오늘의 한 곳 + 할일 요약** 하나로 통일해 “한 눈에 끝”이 보이게.

이 방향으로 가면 “뭘 해야 하지?”가 줄고, “이거 하면 되겠다”가 먼저 보여서 사용자가 더 쉽게 행동하게 만들 수 있습니다.
