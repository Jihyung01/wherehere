# 발전 로드맵 vs 현재 구현 상태

대화에서 정리한 항목이 **이미 있는지 / 아직인지** 빠르게 보는 표입니다.

## 1. 인프라·배포

| 항목 | 상태 | 비고 |
|------|------|------|
| 백엔드 재호스팅 | 운영 작업 | 코드 아님 — Render/Cloud Run 등 선택 후 배포 |
| 헬스체크 | **있음** | `GET /health` (FastAPI `main.py`) |
| 프론트에서 백엔드 상태 표시 | **추가됨** | `GET /api/health` (Next) + `BackendStatusBanner` |
| 환경 분리 (Preview URL) | 부분 | Vercel에서 Preview용 `NEXT_PUBLIC_API_URL` 수동 설정 |

## 2. 추천·개인화

| 항목 | 상태 | 비고 |
|------|------|------|
| 역할 ↔ Kakao 카테고리 정합 | **개선됨** | `KAKAO_ROLE_CATEGORY_MAP`로 스코어링 시 Kakao `primary_category`와 매칭 |
| 10가지 기분 반영 | **개선됨** | 한글 라벨 부분 문자열 + 기존 키워드 규칙 병행 |
| Big Five → 추천 가중 | 없음 | `user_personality`는 추천 API 미연동 (프로필/서사 등만) |
| 방문 이력 가중 | **있음** | `category_preferences` (별점 4+ 카테고리 보너스) |
| 날씨/추천 캐시·TTL | **있음** | 인메모리: `WEATHER_CACHE_TTL_SECONDS`(기본 600초), `RECOMMENDATION_CACHE_TTL_SECONDS`(기본 120초). `docs/CACHE_AND_WEATHER.md` 참고 |

## 3. 프론트·UX

| 항목 | 상태 | 비고 |
|------|------|------|
| API 실패 배너 | **추가됨** | 백엔드 `/health` 경유 확인 |
| 오프라인·로컬 백업 | 없음 | localStorage 일부만 (설정 등) |
| 로딩·스켈레톤 통일 | 부분 | 화면별 상이 |
| Android 15 edge-to-edge | 없음 | TWA 템플릿 업데이트 시점에 대응 |

## 4. 소셜·로컬·푸시

| 항목 | 상태 | 비고 |
|------|------|------|
| 카카오 친구·메시지 | **있음** | `LocalHub`, 백엔드 `kakao-friends`, 퀘스트 완료 공유 등 |
| 피드 페이지네이션 | 부분 | API·UI 단계적 개선 여지 |
| 푸시 E2E | 부분 | VAPID·구독 라우트 있음, 운영 테스트 권장 |

## 5. 품질·유지보수

| 항목 | 상태 | 비고 |
|------|------|------|
| `ignoreBuildErrors` 제거 | 없음 | 단계적 TS 정리 필요 |
| 단위 테스트 | 없음 | 프록시·스코어링 등 추가 권장 |
| 배포 체크리스트 | **추가됨** | `docs/DEPLOYMENT_CHECKLIST.md` |

## 6. 수익·운영

| 항목 | 상태 | 비고 |
|------|------|------|
| AI 토큰 상한 문서 | 부분 | `docs/다음_단계_추천_및_AI_토큰_현황.md` 등 |
| 분석 이벤트 | 없음 | Vercel Analytics 등 선택 |

**한 줄:** 이미 구현된 것(헬스, 프록시, 카카오, 방문 보너스 등)은 표에 반영했고, **이번에 코드로 넣은 것**은 헬스 프록시·배너·배포 문서·추천 역할/기분 정합입니다. 나머지는 백엔드 호스팅 복구와 Big Five·캐시·테스트가 다음 묶음입니다.
